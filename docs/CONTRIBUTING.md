# Contributing

Thanks for considering contributing! Please follow these steps:

- Fork the repo and create feature branches (`git checkout -b feat/your-feature`).
- Keep changes small and focused; open issues for larger discussions.
- Run locally before submitting a PR:
  - `pip install -r backend/requirements.txt`
  - `python backend/app.py`
- Follow Python style and linting in your editor. Add tests for new behaviors where applicable.
- Use clear commit messages and reference issues in PR descriptions.

## Reporting bugs
Open an issue with steps to reproduce, expected behavior, and logs (if applicable).
